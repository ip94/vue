Model-View-Controller

A model “manages data, logic, rules” of the application; view is the component that determines the interface; and controller is the element that reacts to changes in user input (Wikipedia). A user sees the view and interacts with controller. Upon a change has been made, the controller receives the changed data, manipulates the model, which then updates the view for the user to see.

Directives

v-model binds to value from user input or component output
v-bind (shorthand = : ) reactively updates HTML
v-for using a for loop to insert html elements. Has a higher priority if combined with v-if.
v-on (shorthand = @) listens to DOM events
v-if attributes that can control insertion or removal of element
{{variable}} substitutes value of variable in html template

Common Hooks

created () Runs when the page is first accessed
mounted () Runs when instance is rendered
updated () Called when data change caused DOM to re-render and patch

My Project Ideas

Using an available API, the Bikeshare API in this case, I wanted to create an interactive one-page application that demonstrates the implementation of basic Vue.js directives and libraries.

1. Design the basic page-view and objective

TITLE
Left half: information about station where you retrieve your bike Right half: information about station where you’ll park
What can I do with these information?

2. Figure out what information is available and what information I need

From the Bikeshare API, I found that there are two databases that are useful for me – one that lists out all the bike stations in Toronto with information such as location, payment method, types of station; the other one lists out the bike and dock information for each station.

3. Design project structures

Project structures are decided upon hierarchy of parents and children components. These require time to think through how the data changes are going to manipulate the model. Where does the large-scale data get split up and used in components? How are the data retrieved and how will it be communicated? Where will it be used? Are there any repetitions?
